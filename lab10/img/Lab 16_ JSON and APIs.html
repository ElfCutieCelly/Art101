Big Idea

Working with your partner, experiment with processing JSON from an API.

Task 1: Create an index.html for your lab

As always, create the proper folders and an index, css, and JavaScript files for this lab. And make sure you also put a link from your art101 homepage to this lab

  1. Create an index.html with three organized sections: Challenge, Problems, and Results
  2. Use heading, div, and paragraph tags to organize your page
  3. As usual, link a JavaScript file from the <head> section of your HTML like this:

        <head>
            <title>Lab 16: JSON and APIs (XKCD comic!)</title>
            <script src="js/lab16.js" defer></script>
            <script src="https://code.jquery.com/jquery-3.4.1.min.js"></script>
        </head>

  4. Make sure you are loading the latest version of jQuery.
  5. Give your index page an output div.
  6. Add a CSS stylesheet in your css folder and link from index.html
  7. Add a link to your JavaScript file in your index page

You don't have to provide a screenshot for this.

Task 2: Read the Docs

You're going to be creating a web page for XKCD. Read the API docs at https://xkcd.com/json.html.  Links to an external site.Start by looking over anything like "Getting Started," "Tutorial,"  "How To Use," or "Examples."

Turns out XKCD is pretty simple and so the docs are minimal.

Task 3: Test Your API Endpoint

Next step is to test your endpoint from the docs. Hit the endpoints in your browser and see what kind of data they return. You will use the data from the endpoint to understand how you will use it in your application.

When viewing JSON, I think it helps to have the "JSON Formatter" chrome extension installed.

Include a screenshot of the endpoint data in your browser.

Task 4: Create a JavaScript file

Create a new javascript file in your code editor in your lab's js folder

  1. Put a comment block Links to an external site. at the top of your program Links to an external site.
  2. Use $.ajax() to retrieve data and store this object in a variable comicObj. 
  3. Within the success callback function, process the parts and add them to your webpage with the appropriate tags.
    1. Give the section a title using the comicObj.title key from the object
    2. Create an image tag using the comicObj.img key from the object
    3. Give the image an alt and title attributes using the comicObj.alt key from the object

Here's a jQuery .ajax recipe to use:

// Using the core $.ajax() method
$.ajax({
    // The URL for the request (from the api docs)
    url: "https://yourapiendpoint.com/",
    // The data to send (will be converted to a query string)
    data: { 
            // here is where any data required by the api 
            //   goes (check the api docs)
            id: 123,
            api_key: "blahblahblah",
          },
    // Whether this is a POST or GET request
    type: "GET",
    // The type of data we expect back
    dataType : "json",
    // What do we do when the api call is successful
    //   all the action goes in here
    success: function(data) {
        // do stuff
        console.log(data);
    },
    // What we do if the api call fails
    error: function (jqXHR, textStatus, errorThrown) { 
        // do stuff
        console.log("Error:", textStatus, errorThrown);
    }
})

Remember:

  • Use the JavaScript console to see if anything went wrong
  • Experiment with new commands in the JavaScript console to see how they work
  • Add comments to your function to say what it does
  • Make it neat using lots of whitespace.
  • Style in programming matters, so use indentations and spaces where necessary.
  • Remember to put a semicolon at the end of every line. 

Include the source of your JavaScript file at the end of your submission.

As usual,

  1. Test your HTML locally
  2. Fix any problems with your code.
  3. When it is working, take a screenshot.
  4. Upload your entire art101 file structure
  5. Test it on the server.
  6. When it is working, take a screenshot.

Remember:

  • Use the console to see what went wrong.
  • Use console.log() to print out variables to see what's happening behind the scenes.
